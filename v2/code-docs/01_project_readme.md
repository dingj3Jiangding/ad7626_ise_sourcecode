# 01 项目入口文档解读

源文件：projects/ad762x_fmc/Readme.md

## 1. 这个文件解决什么问题

这是工程定位文件，不是实现文件。它告诉你这个工程对应哪条硬件链路、哪些 ADC 型号共用该 HDL。

## 2. 关键代码片段 + 解释

```markdown
# CN0577 HDL Project

* Parts: AD7626
* Parts: AD7625
* Parts: AD7961
* Parts: AD7960
```

解释：

- 说明 ad762x_fmc 不是只服务 AD7626，而是一个家族工程。
- 迁移时如果你只做 AD7626，要明确只保留 16-bit 路径，不要把 18-bit 场景默认带进来。

```markdown
* HDL Doc: .../cn0577/hdl
* Project Doc: .../cn0577
```

解释：

- 这两条外链是系统级背景资料入口。
- 你写新板驱动时，时序和接口行为要以 HDL 实际代码为准，wiki 更多是功能和连接关系参考。

## 3. 迁移前你应得出的结论

1. 当前工程是 CN0577 参考设计，默认目标平台不是 Spartan-6。
2. 代码层迁移要聚焦 AD7626 的单通道差分采样路径。
3. 必须把平台耦合部分（Zynq PS7、HP 端口）从主链路认知里剥离。

## 4. 阅读本文件时的检查点

1. 是否确认目标器件就是 AD7626（16-bit）。
2. 是否确认你需要的是 HDL 采样链路而不是 Linux 驱动端细节。
3. 是否接受“同一工程多器件共用”的事实并做范围收敛。
