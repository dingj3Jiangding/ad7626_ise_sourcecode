# 16 总纲文档使用说明

源文件：projects/ad762x_fmc/common/ad7626_spartan6_driver_guide.md

## 1. 为什么还需要这份文档

你已经有逐文件代码详解，但真正做迁移时还需要一份“跨文件执行路线图”。

这份文档的作用是：

1. 把多文件知识串成迁移步骤
2. 标出可复用和必须替换边界
3. 给出最小 bring-up 顺序

## 2. 建议阅读方式

1. 先读本目录 02 到 10，建立代码细节。
2. 再读总纲文档中的迁移策略与步骤。
3. 最后回到本目录 11 到 15，把寄存器流程落到驱动代码。

## 3. 总纲里最值得反复看的三段

1. 复用与替换边界
2. 新板驱动启动顺序
3. 风险点与排障优先级

## 4. 你可以把总纲转成执行清单

示例执行清单：

1. 完成 Spartan-6 原语替换（ad_data_in 路径）。
2. 完成新 top 与约束。
3. 完成最小寄存器驱动（CLKGEN/PWM/ADC/DMA）。
4. 完成一次 buffer 搬运并验证样本稳定。

## 5. 与本目录文档的关系

- 总纲：做什么、先后顺序、风险管理。
- 逐文件文档：怎么做、代码怎么读、每行是什么意思。

两者配合使用，迁移效率最高。
