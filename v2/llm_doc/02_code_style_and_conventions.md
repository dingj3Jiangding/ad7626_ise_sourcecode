# 代码风格与协作约定

## 1. HDL 风格

- 默认使用 Verilog-2001 语法。
- 参数化优先，避免硬编码常数。
- 状态/控制信号命名要语义清晰，例如：
  - `frame_start`、`bit_tick`、`sample_valid`、`align_error`。
- 调试信号用 `_dbg` 后缀。
- 时序逻辑统一用非阻塞赋值。

## 2. ISE 兼容性约定

- 避免不必要的 SystemVerilog 特性。
- 对位宽推导不依赖 `$clog2`，优先显式参数（如 `BIT_COUNT_W`）。
- Testbench 里失败机制尽量使用 ISE 可稳定支持的写法。

## 3. 注释与文档风格

- 注释和说明文档优先中文，便于团队快速沟通。
- 文档采用“代码片段 + 逐段解释”结构，避免只写概念。
- 每个模块文档应包含：
  1. 关键接口代码片段
  2. 关键时序/数据路径片段
  3. 参数影响说明
  4. 常见错误与调参提示

## 4. 目录协作规则

- 新模块先放 `v2/dev/rtl`。
- 对应 testbench 放 `v2/dev/tb`。
- 对应仿真脚本放 `v2/dev/sim`。
- 对应讲解文档放 `v2/dev/doc`。

## 5. 变更原则

- 小步提交，优先保证最小可验证闭环。
- 先通过仿真再进入板级联调。
- 不改动 `v2/ref` 内容，必要时仅引用/对照。
